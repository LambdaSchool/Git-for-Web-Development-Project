    1. What is Semantic HTML?
	Semantic HTML is to give order to web pages/code. It is stuff like <section>, <main>, <header>, <aside>, <footer>, etc.
	It tells browsers what each portion of a web page is for and what it is.
    2. What is HTML used for?
	Hypertext Markup Language. Used to mark up, organize, and structure the content of web pages.
    3. What is an attribute and where do we put it?
	The attribute typically goes inside the opening tag of an element and gives more information about the element.
    4. What is the h1 tag used for? How many times should I use it on a page?
	The h1 tag is used to create large headings. Use one time per page.
    5. Name two tags that have required attributes
	<form> & <input>
    6. What do we put in the head of our HTML document?
	Meta data such as title, name, authors, description, keywords, viewport information, and emoticon for tabs.
    7. What is an id?
	A unique hash identifier for a particular element; it is placed in the opening tag.
    8. What elements can I add an id to?
	Unique things that you don't want to be paired with anything else. <div>, <span>, <p>, <h#>. You can add one to anything.
    9. How many times can I use the same id on a page?
	One time.
    10. What is a class?
	A unique identifier for one or more elements.
    11. What elements can I add a class to?
	<div>, <span>, <p>, <h#>. You can add one to anything.
    12. How many times can I use the same class on a page?
	No limit.
    13. How do I get my link to open in a new tab?
	<a href="#" target="newpage.html">
    14. What is the alt attribute in the image tag used for?
	The alt is used to display meta data about the image in the case that the browser cannot render the image or for accessibility
	purposes for those that are blind.
    15. How do I reference an id?
	Use a hash in CSS, then the id name.
    16. What is the difference between a section and a div?
	A section partitions the web page into smaller portions for the web browser to understand the content more fully. A <div>
	is used mainly for styling in CSS; <div>s are not displayed.
    17. What is CSS used for?
	To style a web page and make it responsive and beautiful.
    18. How do we select an element? Example - every h2 on the page?
	h2
    19. What is the difference between a class and an id? - Give me an example of when I might use each one?
	Class is for selecting multiple elements, an id is for one unique element. Class could be used for certain things
	you want styled the same, and an id is for something specific.
    20. How do we select classes in CSS?
	.class
    21. How do we select a p element with a single class of “human””?
	p .human
    22. What is a parent child selector? When would this be useful?
	 Parent-child selector is like the p .human as mentioned above. It is useful to select a very specific thing in an already
	established grouping.
    23. How do you select all links within a div with the class of sidebar?
	.sidebar a 
    24. What is a pseudo selector?
	It is used to identify a unique state of an element, such as when one hovers over text, or clicks inside a box.
    25. What do we use to change the spacing between lines?
	line-height:
    26. What do we use to change the spacing between letters?
	letter-spacing:
    27. What do we use to to change everything to CAPITALS? lowercase? Capitalize?
	text-transform: uppercase; text-transform: lowercase; text-transform: capitalize;
    28. How do I add a 1px border around my div that is dotted and black?
	div {
		border: 1px;
		border-style: dotted;
		color: black;
	}
    29. How do I select everything on the page?
	*
    30. How do I write a comment in CSS?
	/* text */
    31. How do I find out what file I am in, when I am using the command line?
	pwd
    32. Using the command line - how do I see a list of files/folders in my current folder?
	ls
    33. How do I remove a file via the command line? Why do I have to be careful with this?
	rm <file name>; we have to be careful because there is no "are you sure?" prompt; you could also use -f to force delete, or
	-r to remove a whole directory. 
    34. Why should I use version control?
	This is important because it allows one to see the history of changes that I have made to a program or website. It helps 
	anyone, the coder themselves or someone else reviewing their code to go back to a previous state of the app/site before 
	an error occurred, assuming that I put sufficient notes so I know what was going on in the previous versions. 
    35. How often should I commit to github?
	Every time I make a change to any of the files in the directory.
    36. What is the command we would use to push our repo up to github?
	git push <remote> <repo>
    37. Walk me through Lambda's git flow.
	Fork the repo, add TL as collaborator, clone the repo, create a branch, add
