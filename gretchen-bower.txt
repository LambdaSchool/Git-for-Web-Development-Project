    37. Walk me through Lambda's git flow. 
    1. **FORK** the repo on Github
2. **CLONE** in VS Code git-cmd use *`git clone url`*
3. **BRANCH** in VS Code git-cmd use *`git checkout -b 'firstName-lastName'`  —* **-b means new branch**
4. Make your changes and add your files
5. **COMMIT** your changes in VS Code git-cmd use *`git add .`*
    1. Confirm branch is ready to commit in VS Code git-cmd with *`git status`*
    2. use *`git commit -m 'be specific'`  —-* **-m means message**
    3. Confirm commit succeeded in VS Code git-cmd with *`git status`*
6. **PUSH** your commit to Github in VS Code git-cmd use *`git push origin -u firstName-lastNam e`*
7. **PULL** to the main branch in Github by clicking *Pull Request* tab and *Merge Pull Request*
